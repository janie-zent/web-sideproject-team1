# Git 워크플로우 규칙

## 커밋 규칙
- **팀원은 커밋하지 않음** — Team Lead가 확인 후 커밋 결정
- 커밋 메시지는 항상 **한국어**로 작성
- 작업 내용을 한 줄로 간결하게 요약

## 커밋 메시지 형식
```
타입: 작업 내용 설명
```

### Prefix (작업유형)
| 타입 | 설명 |
|------|------|
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| docs | 문서 수정 |
| style | 코드 포맷팅 |
| refactor | 코드 리팩토링 |
| test | 테스트 코드 추가/수정 |
| chore | 빌드/패키지 관련 |

### 예시
```
feat: 세무 일정 목록 화면 추가
fix: 사용자 조회 API JWT 검증 오류 수정
test: Dexie 스키마 단위 테스트 작성
chore: User 모델 마이그레이션
```

## 브랜치 전략
- `main` 기반으로 작업
- 작업 브랜치: `{영역}/{작업내용}` (예: `frontend/tax-schedule-list`)
- 코어 브랜치(`main`)에 직접 Push 금지
